Componentes da Arquitetura VivoSynq

Cache de Requições

NGINX

O Balance Loader/Cache de requisições irá gerenciar a demanda de requisições em cada container, e armazena um cache das respostas para aliviar o peso dos containers.

API RESTful

GO

API RESTful que apresentará os dados recuperados do cliente. Os dados serão buscados do Banco de dados de CACHE ou do Banco de dados Integrado MongoDB.

Cache de Dados Recentes

Redis

Servidor dedicado para armazenar o CACHE dos produtos do cliente com 15 (quinze) minutos de tolerância.

Base de Dados Integrada

MongoDB

Servidor dedicado para armazenar os dados dos produtos do cliente.

Dashboard de Monitoramento

Grafana/Prometheus

Visualização de Dashboard para monitoramento da saúde da arquitetura. Esse container que consultará o banco de dados integrado para obtenção de status sobre rede, dados e componentes.

Consumidor da Fila de Requisições

Go - confluentic/confluent-kafka-go

Consumidor de filas do Apache Kafka. Recebe os dados alterados das bases da Vivo e reflete as alterações no domínio de produtos.

Fila de Atualizações das Bases Legadas

Kafka/Zookeeper

Fila de atualizações das bases legadas da Vivo. Recebe as alterações do conector (Debezium) e disponibiliza os dados para serem consumidos.

Leitor de Mudanças das Bases Legadas

Debezium

Conector para banco de dados Debezium. Lê as atualizações do banco de dados e as insere na fila do Apache Kafka.
